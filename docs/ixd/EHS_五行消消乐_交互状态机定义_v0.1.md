## 一、完整状态机（文字版状态流转图）

### 1.1 状态定义总览

| 状态编码 | 状态名 | 视觉表现 | 层级归属 |
|---------|--------|---------|---------|
| S0 | 未触碰态 | 显示预切字卡片，无高亮 | 常态层 |
| S1 | 选中态 | 卡片高亮边框+微放大(105%) | 常态层 |
| S2 | 质心扩展态 | 卡片放大150%+底稿方圆透出+扩展框 | 扩展层 |
| S3 | 拖动对齐态 | 底层字跟随手指+切割框反向映射 | 扩展层 |
| S4 | 围盒精修态 | 进入笔模式，可绘制闭合路径 | 精修层 |
| S5 | 入库动画态 | 字块飞入字库+冲屏特效 | 特效层 |
| S6 | 待定下沉态 | 字块下沉+灰度标记 | 常态层 |
| SE | 异常回弹态 | 软边界回弹动画 | 全局 |

## 二、手势映射表

### 2.1 核心手势定义

| 手势 | 触发条件 | 系统反馈 | 适用状态 | 触发后状态 |
|------|---------|---------|---------|-----------|
| **单击** | 手指按下+抬起，停留<200ms，位移<10dp | 选中高亮：卡片边框2dp蓝色光晕+微放大105% | S0 | S1 |
| **上滑** | 按下→向上滑动，Y位移>60dp，角度偏差≤30°（相对垂直轴） | 字块脱离网格，沿抛物线飞入顶部字库栏，触发冲屏特效 | S1 | S5 |
| **下拉** | 按下→向下滑动，Y位移>60dp，角度偏差≤30°（相对垂直轴） | 字块下沉30dp+叠加30%灰度遮罩，显示"待定"角标 | S1 | S6 |
| **长按** | 单指持续按压≥500ms，位移<15dp | 卡片放大至150%，底稿原图方圆区域渐显，出现扩展切割框 | S1 | S2 |
| **双击** | 两次单击间隔<300ms | 进入围盒精修笔模式，显示虚线画笔光标 | S2, S3 | S4 |
| **拖动** | 长按进入S2后，手指在屏幕上移动 | 底层原稿字跟随手指移动，切割框反向映射（反向偏移） | S2 | S3 |
| **释放** | 拖动后手指离开屏幕 | 以当前手指位置为新质心，重新执行局部分割，刷新显示 | S3 | S2 |
| **双指缩放** | 两根手指在屏幕上做张开/捏合动作 | 底稿放大率变化：1.0x→3.0x，单指/松手恢复 | 任意态 | 局部放大 |

### 2.2 手势参数精确值

| 参数项 | 数值 | 说明 |
|--------|------|------|
| 单击最大停留时间 | 200ms | 超过则判定为长按意图 |
| 单击最大位移 | 10dp | 超过则不触发单击 |
| 选中自动取消延时 | 300ms | S1状态无操作自动回S0 |
| 长按触发阈值 | 500ms | 持续按压达到此值进入S2 |
| 长按最大位移容差 | 15dp | 按压期间位移超此值取消长按 |
| 滑动触发Y位移 | 60dp | Y轴位移达到此值才判定上滑/下拉 |
| 滑动角度容差 | 30° | 相对垂直轴的偏差角度，超过则不触发消除 |
| 双击最大间隔 | 300ms | 两次点击间隔超此值不判定双击 |
| 拖动实时反馈率 | 60fps | 底层字跟随的刷新帧率 |
| 扩展放大比例 | 1.5x | S2态的卡片放大倍数 |

### 2.3 手势冲突消解规则

```
手势识别优先级（高→低）：
1. 双指缩放（最优先，任意态识别）
2. 双击（仅S2/S3态识别，优先于其他单指手势）
3. 长按（S1态，需持续500ms，优先于滑动）
4. 上滑/下拉（S1态，角度判定后触发）
5. 单击（兜底识别，无其他手势匹配时触发）

冲突处理：
- 长按期间检测到位移>15dp → 取消长按计时，视为普通拖动（无状态变更）
- 上滑/下拉过程中检测到角度偏差>30° → 中断消除流程，回S0
- S2态下短按（<200ms）→ 视为"单击空白"，忽略（保持S2）
- S4笔模式下，单指绘制路径时需关闭其他手势识别器
```

## 三、双层卡片交互细节

### 3.1 三层架构定义

│  z-index: 30                                                 │
├─────────────────────────────────────────────────────────────┤
│                      【扩展层 S2/S3】                        │

│  z-index: 20                                                 │
├─────────────────────────────────────────────────────────────┤
│                      【常态层 S0/S1/S5/S6】                  │
│  消消乐主界面（预切字卡片+网格布局+字库栏）                   │

### 3.2 常态层（S0/S1/S5/S6）

#### 3.2.1 视觉规范
- 卡片尺寸：W=80dp × H=80dp（标准字块）
- 卡片圆角：8dp
- 卡片背景：白色 #FFFFFF，阴影 elevation=2dp
- 预切字显示：黑色毛笔字，居中对齐，字约占卡片70%面积
- 网格间距：cardGap=8dp

#### 3.2.2 S0→S1 选中切换
- 触发：单击卡片
- 动效序列：
  1. t=0ms：边框出现（2dp，颜色=#2196F3 蓝）
  2. t=0~100ms：Scale 1.0→1.05（easeOutQuad）
  3. t=100~150ms：微弹簧回弹至1.05稳定
- 取消：300ms内无操作自动回S0（反向动画）

#### 3.2.3 S1→S5 上滑入库
- 触发：上滑手势（角度≤30°）
- 动效序列见【第五章 5.1 冲屏特效6步流程】

#### 3.2.4 S1→S6 下拉待定
- 触发：下拉手势（角度≤30°）
- 动效序列：
  1. t=0~150ms：卡片向下平移30dp（easeOutQuad）
  2. t=100~200ms：叠加灰度遮罩（灰度值30%，渐显）
  3. t=150~300ms：右下角出现"待定"角标（16dp圆形，灰底白字）
- 再次单击S6卡片→恢复常态（反向动画150ms）

### 3.3 扩展层（S2/S3）

#### 3.3.1 S1→S2 质心扩展切换
- 触发：长按0.5s
- 完整切换序列：

```
Step 1 [0~100ms]:   卡片继续放大 1.05→1.5（easeOutCubic）
Step 2 [0~200ms]:   底稿层渐显 alpha 0→0.6（线性）
                    预切字层渐隐 alpha 1→0.3（线性）
Step 3 [100~300ms]: 切割框出现（虚线方框，线宽1.5dp，#FF9800 橙色）
                    框大小=字块原始切割框×1.2（留出调整空间）
Step 4 [200~300ms]: 出现操作提示：
                    - 顶部提示条："拖动对齐底层字，双击进入精修"
                    - 提示条 auto-dismiss 3000ms
```

#### 3.3.2 S2→S3 拖动对齐
- 触发：手指在S2卡片上移动
- 交互逻辑：
  - 底层原稿（底稿层）**正向跟随**手指移动
  - 预切字（常态层）**反向映射**：切割框移动方向与手指相反
  - 实时反馈：60fps刷新
  - 视觉辅助：手指位置显示十字准星（横竖虚线贯穿屏幕）

#### 3.3.3 S3→S2 释放重切
- 触发：手指离开屏幕
- 处理流程：
  1. 记录释放点坐标 (releaseX, releaseY)
  2. 计算偏移量：offset = (releaseX - originX, releaseY - originY)
  3. 传递给Layer3：调用OpenCV局部分割，以新质心重新切割
  4. 等待回调（≤100ms）：刷新预切字显示
  5. 视觉反馈：新字块闪烁一次（alpha 0.5→1.0，100ms）

### 3.4 精修层（S4）

#### 3.4.1 S2/S3→S4 进入围盒精修
- 触发：双击
- 切换序列：
  1. t=0ms：隐藏扩展层操作提示，显示画笔光标（圆形，12dp，#FF9800）
  2. t=0~100ms：底部弹出工具栏（撤销/重做/确认/取消）
  3. 背景蒙版：底稿层alpha提升至0.8，其他卡片半透明遮罩

#### 3.4.2 S4 路径绘制逻辑
- 绘制方式：手指拖动绘制自由路径
- 路径可视化：
  - 线条：2dp，虚线样式，颜色#FF9800（橙）
  - 锚点：路径拐点显示小圆点（4dp）
  - 闭合判定：终点与起点距离<20dp时自动吸附闭合
- 闭合后：
  - 闭合区域填充半透明橙色（alpha=0.2）
  - 显示"确认裁剪"按钮（底部工具栏主按钮高亮）

#### 3.4.3 S4→S2 确认裁剪
- 触发：点击"确认裁剪"或闭合路径后自动弹出确认
- 处理流程：
  1. 将闭合路径坐标数组传递给Layer3
  2. Layer3执行OpenCV掩码裁剪（mask-based crop）
  3. 回调刷新：新裁剪字块替换原预切字
  4. 视觉反馈：新字块从裁剪区域放大弹出（scale 0.5→1.0，200ms，easeOutBack）
  5. 返回S2态（保持扩展层，可继续调整或单击退出）

#### 3.4.4 S4→S2 取消/退出
- 触发：点击"取消"、手势返回（左滑边缘）、单击蒙版空白区域
- 处理：丢弃当前路径，无裁剪操作，150ms渐隐返回S2

## 四、与Agent-D的接口：Compose State定义

### 4.1 核心状态类（Kotlin Data Class）

```kotlin
/**
 * 五行消消乐核心状态机 — Compose State定义
 * 所有状态变更通过ViewModel + MutableStateFlow驱动UI重组
 */

// ═══════════════════════════════════════════════
// 根状态：消消乐游戏板状态
// ═══════════════════════════════════════════════
data class CharacterBoardState(
    val characters: List<CharacterBlock>,  // 字块列表
    val boardMode: BoardMode = BoardMode.NORMAL,  // 全局模式
    val selectedId: String? = null,  // 当前选中字块ID
    val activeLayer: ActiveLayer = ActiveLayer.BASE,  // 活跃层级
    val resourceBar: ResourceBarState = ResourceBarState(),  // 资源栏（经验/灵石）
    val pendingCount: Int = 0,  // 待处理字块计数
    val storedCount: Int = 0,   // 已入库字块计数
)

// ═══════════════════════════════════════════════
// 单个字块状态（核心）
// ═══════════════════════════════════════════════
data class CharacterBlock(
    val id: String,                    // 唯一标识：行号_序号（如 "3_05"）
    val ocrText: String,               // OCR识别文本
    val preCutBitmap: Bitmap?,         // Layer3预切字位图（常态显示）
    val rawBitmap: Bitmap?,            // 底稿原始位图（扩展层透出）
    val cutBox: RectF,                 // 当前切割框（相对坐标）
    val characterState: CharacterState = CharacterState.IDLE,  // 字块状态
    val elementType: ElementType?,     // 五行属性（金木水火土），null=未识别
    val confidence: Float = 0f,        // OCR置信度 0~1
    val revisionHistory: List<CutRevision> = emptyList(),  // 裁剪历史（Git式）
)

// 字块状态枚举（对应状态机S0~S6）
enum class CharacterState {
    IDLE,           // S0: 未触碰
    SELECTED,       // S1: 选中态
    EXPANDED,       // S2: 质心扩展态
    DRAGGING,       // S3: 拖动对齐态
    REFINING,       // S4: 围盒精修态
    STORING,        // S5: 入库动画态（进行中）
    PENDING,        // S6: 待定下沉态
}

## 五、动效时序

### 5.1 冲屏特效6步流程（上滑入库S1→S5）

> 触发条件：选中字块后上滑手势，角度偏差≤30°
> 总时长：600ms

```
Step 1 [0~80ms]     字块脱离
  ├─ 卡片从网格中"浮起"，elevation 2dp→16dp
  ├─ Scale 1.05→1.15（轻微放大表示激活）
  ├─ 网格中该位置留出空白（占位符半透明闪烁）
  └─ Easing: easeOutQuad

Step 2 [80~200ms]   抛物线加速
  ├─ 字块沿贝塞尔曲线飞向顶部字库栏
  ├─ 曲线控制点：向上凸起30%（拱形轨迹）
  ├─ Scale 1.15→0.8（渐小，模拟远去）
  ├─ Rotation: 0°→15°（轻微旋转增加动感）
  ├─ 字块拖尾：生成3个残影副本（间隔20ms，alpha递减）
  └─ Easing: easeInCubic（加速感）

Step 3 [200~280ms]  字块入栏
  ├─ 字库栏对应槽位高亮闪烁（#FFD700 金色，alpha 0→0.5→0）
  ├─ 字块scale 0.8→1.0（入栏后恢复）
  ├─ 字块rotation 15°→0°（恢复正向）
  ├─ 字库计数器数字跳动动画（+1，上浮10dp后消失）
  └─ Easing: easeOutBack（轻微回弹）

Step 4 [280~400ms]  冲屏特效（五行爆发）
  ├─ 根据字块五行属性触发对应颜色粒子爆发：
  │   金:#FFD700 木:#4CAF50 水:#2196F3 火:#F44336 土:#795548
  ├─ 粒子数：20~30个（根据屏幕密度适配）
  ├─ 粒子形状：圆形+星形混合
  ├─ 粒子运动：向外辐射扩散，半径100~150dp
  ├─ 粒子生命周期：120~200ms（随机）
  └─ 背景微闪：全屏overlay对应五行色，alpha 0→0.1→0

Step 5 [400~520ms]  资源飞字
  ├─ "经验+10" 文字从字库栏飘出（Y轴上浮40dp）
  ├─ "灵石+1"  文字延迟40ms跟随飘出
  ├─ 文字样式：16sp，金色加粗，带2dp黑色描边（确保可读性）
  ├─ Alpha: 1.0→0.0（渐隐）
  ├─ 经验条进度条填充动画（对应增量部分）
  └─ 如有境界突破，触发突破预备（见5.2）

Step 6 [520~600ms]  网格复位
  ├─ 原位置空白占位符淡出（alpha 0.3→0，80ms）
  ├─ 下方字块上移填补空缺（级联动画，每格延迟30ms）
  ├─ 新字块从顶部滑入（如有滚动加载）
  ├─ 整体网格轻微震动反馈（vibration 50ms）
  └─ 状态回归S0，等待下一次交互
```

### 5.2 突破动画5步流程（境界提升）

> 触发条件：经验值累计达到当前境界上限
> 总时长：2500ms（含预备等待）

```
Step 1 [0~200ms]     突破预警
  ├─ 经验条填满瞬间：条颜色变为金色闪烁（3次，间隔60ms）
  ├─ 屏幕四周出现金色光晕边框（向内收缩）
  ├─ 背景BGM切换为突破专用音效
  ├─ 所有字块微微震动（振幅2dp，频率8Hz，持续200ms）
  └─ 显示"突破契机！"文字（屏幕中央，放大入场）

Step 2 [200~800ms]   灵气汇聚
  ├─ 屏幕四周向中央汇聚金色光线（8条，旋转对称）
  ├─ 光线宽度：2dp→8dp（渐粗）
  ├─ 中央形成金色旋涡（旋转动画，2转/秒）
  ├─ 旋涡中心亮度递增（alpha 0.2→0.8）
  ├─ 已入库的字块虚影从字库栏飘出，围绕旋涡旋转
  └─ 音效：风声→雷鸣渐变

Step 3 [800~1500ms]  破境冲关
  ├─ 旋涡中心爆发白光（全屏overlay，alpha 0→0.6→0，200ms）
  ├─ 屏幕震动效果（振幅5dp，持续300ms）
  ├─ 出现裂纹特效（屏幕边缘向内"裂开"，金色裂纹纹理）
  ├─ 境界文字动画：旧境界碎裂→新境界组合
  │   旧境界：碎成粒子散开
  │   新境界：从旋涡中心凝聚成型
  └─ 显示新境界名称（如"炼气一阶→炼气二阶"）

Step 4 [1500~2200ms] 天道回馈
  ├─ 爆发消散，金色粒子如雨落下（全屏粒子效果）
  ├─ 新境界文字高亮停留（金色发光效果，shadow blur=8dp）
  ├─ 资源奖励批量飞入：
  │   灵石+50（主奖励，大字号，金色）
  │   经验上限提升（提示文字）
  ├─ 经验条清空并重新填充（新境界的初始进度）
  └─ 字库栏升级外观（如有样式变化）

Step 5 [2200~2500ms] 收束回归
  ├─ 所有特效渐隐（alpha→0，300ms）
  ├─ 境界文字缩小移至顶部状态栏固定位置
  ├─ 屏幕恢复正常交互
  ├─ 播放突破完成音效

## 六、异常流完整定义

### 6.1 误触自动取消
- 条件：S1选中态下，300ms内无任何手势输入
- 流程：自动触发deselectCharacter()→回S0
- 视觉：高亮边框渐隐（100ms），scale恢复1.0

### 6.2 角度偏差拒绝
- 条件：S1态下触发滑动，但相对垂直轴角度>30°
- 流程：不触发消除/待定，视为普通页面滚动
- 视觉：字块轻微跟随手指移动（粘性效果），松手后弹性回归原位（200ms，easeOutElastic）

### 6.3 多指缩放优先
- 条件：任意状态下检测到双指触摸
- 流程：暂停当前手势识别，切换为缩放模式
- 视觉：底稿放大，显示当前缩放比例指示器（屏幕右上角，2秒后淡出）
- 恢复：双指离开→缩放指示器淡出→恢复单指手势识别

### 6.4 拖动越界回弹
- 条件：S3拖动态下手指坐标超出safe area
- 流程：见【5.3 软边界回弹动效】
- 特殊：若手指在越界位置直接抬起→按越界前最后有效坐标执行重切

### 6.5 精修路径未闭合
- 条件：S4态下单击"确认"但路径未闭合
- 流程：拒绝确认，显示提示"请先绘制闭合区域"
- 视觉：路径线条闪烁红色（2次），底部按钮保持未激活态

### 6.6 OpenCV切割超时
- 条件：releaseCharacter()后Layer3回调超过500ms未返回
- 流程：显示loading指示器（卡片中央，旋转圆环），超时1000ms后显示"切割中..."提示
- 恢复：回调到达后刷新显示；若报错→保持原状+Toast提示"切割失败，请重试"
