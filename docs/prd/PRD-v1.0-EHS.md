## 1. 产品概述

### 1.1 一句话定位

**个人笔迹数字孪生修仙游戏。** 用户扫描历史手写稿，通过五行消消乐校对构建Git版本字库，生成手写风格文档。

### 1.2 产品形态

移动端应用（Android优先），单机关卡制游戏+个人字库工具的二元融合体。以修仙文化为世界观包装，将枯燥的OCR校对过程转化为五行消消乐玩法，将字库积累过程转化为修仙境界成长，将最终输出包装为红尘历练的万象法相。

### 1.3 核心价值主张

- **对种子用户（8-18岁学生）**：把作业/日记/作文的字迹变成游戏角色，玩消消乐就是在"修炼自己的字体"，输出手写体文档在同学间形成社交货币。
- **对付费用户（22-60岁成年人）**：将多年手写笔记、情书、家书数字化为个人专属字体，用于生成有温度的个性化文档（请柬、家书、诗词），替代冷冰冰的印刷体。
- **对平台**：以游戏化降低OCR数据标注成本，以UGC字库构建护城河。

### 1.4 目标用户画像

#### 1.4.1 种子用户：中小学生群体（8-18岁）

| 维度 | 描述 |

| 字迹特征 | 正在成型期，书写习惯尚未固化，同一字存在较大差异 |
| 痛点 | 字迹被批评"潦草"；想有好看的手写体但练字枯燥；同学间炫耀个性化内容 |
| 诉求 | 游戏化体验、成就感、社交展示、免费优先 |
| 付费意愿 | 低，依赖家长付费决策 |
| 触达渠道 | 教育类App合作、家长社群、学校活动 |

| 维度 | 描述 |

## 2. 核心系统架构（总览）

EHS由七大核心系统构成，形成"扫描-校对-积累-输出"的完整闭环。

```
┌─────────────────────────────────────────────────────────────────┐
│                        蚯蚓.手书修仙传                           │
│                    Eruca HandScript: Cultivation Saga            │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  ┌──────────┐    ┌──────────────┐    ┌──────────────────────┐  │
│  │ 挖矿系统  │───▶│ 五行消消乐    │───▶│     金字招牌系统      │  │
│  │ (扫描OCR)│    │ (核心玩法)    │    │  (Git分支+爆屏特效)   │  │
│  └──────────┘    └──────┬───────┘    └──────────┬───────────┘  │
│                         │                       │                │
│                         ▼                       ▼                │
│                  ┌──────────────┐    ┌──────────────────────┐  │
│                  │  修仙境界     │    │     五行灵根          │  │
│                  │(炼气→飞升)   │    │   (笔迹命格报告)      │  │
│                  └──────┬───────┘    └──────────┬───────────┘  │
│                         │                       │                │
│                         └───────────┬───────────┘                │
│                                     ▼                            │
│                          ┌──────────────────────┐               │
│                          │     万象法相          │               │
│                          │ (个性化文档输出)       │               │
│                          └──────────┬───────────┘               │
│                                     │                            │
│                          ┌──────────▼───────────┐               │
│                          │     天道功德(V2)      │               │
│                          │  (区块链确权+付费)    │               │
│                          └──────────────────────┘               │
│                                                                 │
├─────────────────────────────────────────────────────────────────┤
│  技术底座：PaddleOCR + OpenCV + 端侧Git版本控制 + Android Native  │
└─────────────────────────────────────────────────────────────────┘
```

**系统间数据流：**

1. **挖矿系统**（手稿扫描）产出原始字图 → 进入**五行消消乐**进行校对
2. **五行消消乐**产出经过确认的字图 → 按字分支部存储，进入**金字招牌**候选池
3. **金字招牌**通过算法+人工精修筛选出标准字形 → 注入**修仙境界**字库
4. **修仙境界**解锁字数阈值后 → 触发**五行灵根**命格报告
5. **五行灵根**属性标签 → 影响**万象法相**输出风格
6. **万象法相**输出时缺字 → 回流至**五行消消乐**进行"缺字补天"
7. **天道功德**(V2) 对金字招牌进行链上确权 → 开放付费输出

## 5. 核心系统详细设计

### 5.1 五行消消乐（核心玩法）

#### 5.1.1 游戏机制

**基础规则：**
- 每次扫描一页手稿，OCR检测出的字块以双层卡片形式从屏幕上方掉落
- 卡片按五行属性（木/火/土/金/水）自动分类着色
- 玩家通过手势操作字块，完成消除与校对

**五行属性映射规则（字→五行）：**

| 判定维度 | 木 | 火 | 土 | 金 | 水 |
|----------|----|----|----|----|----|
| 笔画特征 | 连笔多、曲线多 | 笔压重、顿点多 | 结构方正、横平竖直 | 棱角分明、钩多 | 流畅、波状线 |
| 代表部首 | 艹木禾 | 火灬日 | 土山石 | 金钅刂 | 氵冫雨 |
| 示例字 | 林、花、茶 | 炎、照、热 | 地、岩、城 | 钢、剑、锋 | 河、清、雪 |

> **注**：五行属性判定由图像特征+部首偏旁双重规则决定，部首权重60%，笔迹特征权重40%。

**操作手势：**

| 手势 | 动作 | 效果 | 适用状态 |
|------|------|------|----------|
| 单击 | 选中/查看 | 双层卡片切换（预切字↔原稿透出） | 所有卡片 |
| 上滑 | 确认入库 | 字块飞入字库，消除动画，+经验 | 绿色/黄色/红色（按境界解锁） |
| 下拉 | 标记待定 | 字块下沉，标记为"待定"，后续处理 | 所有卡片 |
| 长按(0.5s) | 质心扩展 | 字块放大150%，透出底稿方圆+拖动对齐 | 所有卡片 |
| 双击 | 精修标记 | 标记为"手动精修"，记录调整历史 | 质心扩展模式下 |

**消除规则：**

| 消除类型 | 触发条件 | 得分 | 特效 | 说明 |
|----------|----------|------|------|------|
| 基础消除 | 1个字块上滑入库 | 10灵石 | 白光消散 | 最基础操作 |
| 三同消除 | 3个同五行字块连续上滑 | 100灵石 | 同色光芒爆发 | 基础combo |
| L型消除 | 4个同五行字块L型排列 | 200灵石 | 五行图腾浮现 | 进阶combo |
| 十字消除 | 5个同五行字块十字排列 | 500灵石 | 全屏该属性光效 | 高级combo |
| 相生连击 | 木→火→土→金→水顺序消除 | 基础×3 | 五行流转光环 | 必须严格按相生顺序 |
| 相克惩罚 | 金克木/木克土/土克水/水克火/火克金 | -50灵石 | 血红闪烁+心魔+震动 | 误消时触发 |

**五行生克关系图：**

```
        木 ──→ 火 ──→ 土 ──→ 金 ──→ 水 ──→ 木
        ↑                              │
        └──────── 相生（增益）──────────┘
        
        木 ──✕ 土    火 ──✕ 金    土 ──✕ 水    金 ──✕ 木    水 ──✕ 火
        
        ──→ 相克（惩罚）
```

**心魔干扰机制：**
当玩家触发相克消除时，触发"心魔入侵"事件：
1. 屏幕边缘泛起血红波纹
2. 心魔虚影（黑雾状字形）从屏幕四周向中央聚拢
3. 震动反馈300ms
4. 扣除50灵石
5. 随后3个新掉落字块中必有1个"心魔块"（红色闪烁，无法消除，需用照妖镜净化）

#### 5.1.2 法器系统（MVP两个法器）

**照妖镜：**
- 获取：筑基期解锁，初始赠送3个
- 效果：高亮显示所有心魔干扰块（红色字块），持续5秒
- 使用时机：心魔块堆积、无法消除时
- 冷却：每局可使用次数不限，但需灵石购买（MVP免费）

**炼字炉：**
- 获取：金丹期解锁，初始赠送1个
- 效果：选择3个同五行普通字，合成1个该字的金字招牌候选
- 合成规则：3个普通字→1个候选字（出场率+3，稳定性取平均）
- 限制：每日限用3次（MVP无此限制）

#### 5.1.3 关卡流程

```
1. 扫描手稿 → 2. 行级检测 → 3. 字块掉落 → 4. 消消乐校对 → 5. 经验结算 → 6. 境界检查
```

单局消消乐流程：
1. 玩家选择"吐纳修炼"（扫描新稿）或"打坐悟道"（处理待定字）
2. 字块从屏幕顶部依次掉落，堆叠在底部网格（6列×8行）
3. 玩家操作字块消除，网格满时Game Over
4. 消除结算：总分 = 基础分 + combo加成 + 相生加成 - 相克惩罚
5. 获得经验值，检查是否满足突破条件
6. 突破时播放境界突破动画，解锁新功能

### 5.2 修仙境界系统

#### 5.2.1 七境界定义

| 境界 | 解锁字数 | 经验阈值 | 解锁功能 | 视觉表现 |
|------|----------|----------|----------|----------|
| 炼气期 | 0 | 0 | 基础消消乐（仅绿色可消） | 淡青灵气环绕 |
| 筑基期 | 100 | 300 | 黄色可消，解锁照妖镜 | 脚底出现基座光环 |
| 金丹期 | 500 | 1,500 | 红色可消，解锁炼字炉，开放万象法相 | 丹田处金丹旋转 |
| 元婴期 | 2,000 | 8,000 | 解锁五行相生连击暴击 | 头顶元婴小人浮现 |

#### 5.2.2 经验值计算公式

```
单次消消乐经验 = Σ(每个消除字块的经验) + 连击加成 + 境界加成

每个字块经验:
  绿色(置信度≥0.9): 3 EXP
  黄色(0.7≤置信度<0.9): 2 EXP
  红色(置信度<0.7): 1 EXP

连击加成:
  连续消除无间隔: ×1.5
  相生连击: ×3.0

境界加成:
  越级挑战(高难度关卡): ×2.0
```

#### 5.2.3 突破机制

当经验值达到当前境界阈值时，触发突破：
1. 屏幕渐变暗，中央出现境界图腾
2. 积蓄动画（灵气/光芒向中央聚拢，持续3秒）
3. 玩家点击屏幕"破境"
4. 全屏特效（对应境界的视觉效果，持续5秒）
5. 新境界名称浮现，解锁功能列表展示
6. 经验值清零，进入下一境界积累

**突破失败机制（MVP简化版）：**
- MVP中突破100%成功，无失败惩罚
- V2可引入突破概率+道具辅助

### 5.3 金字招牌系统

#### 5.3.1 版本控制模型

每个汉字拥有一个独立的"版本仓库"，采用简化Git分支模型：

```
字: "永"
│
├── main (标准字形分支)
│   └── v3 (当前标准字形) ◀── 冲屏而出后确立
│
├── dev (开发分支)
│   ├── v1 (2026-06-01 14:32) ── 出场率3，稳定性0.82
│   ├── v2 (2026-06-03 09:15) ── 出场率5，稳定性0.91，未精修
│   └── v3 (2026-06-05 20:47) ── 出场率7，稳定性0.97，已精修 ★达标
│
└── stash (暂存分支)
    └── 待定字块队列
```

#### 5.3.2 三围指标计算

| 指标 | 计算方式 | 达标阈值 | 说明 |
|------|----------|----------|------|
| 出场率 | 该版本在所有同字出现中的计数 | ≥ 5次 | 使用频率足够高 |
| 稳定性 | 1 - (置信度标准差 / 平均置信度) | ≥ 0.95 | OCR识别一致性好 |
| 精修标记 | 用户在质心扩展模式下手动调整并确认 | = true | 人工校验通过 |

**三围达标检查：** 每次消消乐结算后自动遍历所有字的dev分支，检查是否有版本满足三围条件。满足则触发冲屏而出。

#### 5.3.3 冲屏而出特效流程

1. **预警**：屏幕边缘泛起金色波纹，背景音乐渐强
2. **凝聚**：达标字块从字库界面缓缓升起，悬浮屏幕中央
3. **冲屏**：字块急速放大至全屏，3D旋转，金光粒子从字笔画间迸发
4. **铭刻**：字块缩小并铭刻入"金字招牌"榜单， accompanied by 庄重音效
5. **确立**：该版本从dev分支合并至main分支，成为标准字形
6. **庆祝**：全屏撒花，经验值额外+100

#### 5.3.4 金字招牌榜单

- 界面：卷轴式UI，已收录字按拼音排序
- 每个字显示：标准字形大图 + 三围数值 + 收录日期
- 点击字可查看该字所有历史版本（时间轴展示）

## 6. 用户故事

### 6.1 用户故事一：小学生的"字体修仙"之路（种子用户核心路径）

**角色：** 小雯，12岁，六年级学生，字迹潦草常被老师批评，喜欢玩手机游戏。

2. **首次引导**：启动后，修仙世界观动画（3秒）→ 提示"道友，请出示你的手稿" → 教程引导拍摄第一张作文。

## 8. 技术架构摘要

### 8.1 四层处理架构

| 层级 | 技术组件 | 职责 | 输出 |
|------|----------|------|------|
| Layer 1 | PaddleOCR DBNet | 行级文字检测 | 行坐标框数组 |
| Layer 2 | return_word_box | 字框初检（ROI引导） | 初框坐标（不直接抠图） |
| Layer 3 | OpenCV | 局部分割（连通域+分水岭） | 精确单字图（消消乐素材） |
| Layer 4 | 人机协同校对 | 五行消消乐交互 | 确认字图+版本控制数据 |

### 8.2 关键约束

| 约束项 | 要求 | 影响 |
|--------|------|------|
| 目标设备 | 华为Mate30（麒麟990） | 算法需在Android端侧实时运行 |
| 系统版本 | Android 10+ | API级别29+ |
| 包体积 | <50MB | 模型需精简/量化，资源压缩 |
| 网络 | 国内环境，优先端侧 | 不上传原稿，处理完全本地 |
| 开发环境 | iMac/MacAir + Android Studio | 开发工具链 |

### 8.3 数据安全

- 原稿图片：AES-256加密存储本地，密钥设备绑定
- 字库数据：SQLite加密数据库
- 不上传：任何用户手稿/字迹数据均不上传服务器
- 导出备份：可选加密导出至用户指定位置
